uri启动Activity

在运营主导的app中，总是会有很多的web结合native的活动需求， 比如某个活动页点击后要跳转到我们的某个页面- -

如果webView 是我们自己写的， 你可以通过设置WebViewClient 然后重写其 shouldOverrideUrlLoading 这个方法然后

手动分析url， 来实现想要做的事情。。。

但是如果是想直接浏览器上打开这个方法就不行了- -

这时候只能将想跳转的activity的uri 设置成相应的就可以了

关于uri

uri 官方文档

官方文档说它遵循的RFC 2396, 这应该是一个url标准,

url还可以干嘛?

比如你要启动某个系统自带的app 直接发一个Intent 发一个uri 就可以了- -
